DOCTYPE有什么作用？标准模式与混杂模式如何区分？它们有何意义?

  • 告诉浏览器使用哪个版本的HTML规范来渲染文档。

  • DOCTYPE不存在或形式不正确会导致HTML文档以混杂模式呈现。

  • 标准模式（Standards mode）以浏览器支持的最高标准运行；

  • 混杂模式（Quirks mode）中页面是一种比较宽松的向后兼容的方式显示。

HTML5为什么只需要写 <DOCTYPE html>？

  • HTML5不基于SGML（Standard Generalized Markup Language 标准通用标记语言）

  • 因此不需要对DTD（DTD 文档类型定义）进行引用

  • 但是需要DOCTYPE来规范浏览器行为

  • HTML4.01基于SGML，所以需要引用DTD,才能告知浏览器文档所使用的文档类型

行内元素有哪些？块级元素有哪些？空(void)元素有那些？

(1) 行内元素

span,a,b,strong,i,em,br,ins,u,del,s,inut

  • 本身属性为 display:inline;
  • 不可以直接设置宽度和高度，宽度和高度是由本身内容大小决定的(文字，图片等)
  • 可以直接设置内外边距的左右值
  • 只能容纳文本或者其他行内元素，不能嵌套块级元素
  • 用于不指定宽高，宽高由内容指定；

(2) 块状元素

div,h1-h6,hr,menu,ol,ul,dl,table,p,form

  • 本身属性为 display:block;
  • 可以直接控制宽度高度，padding，margin等
  • 不设置宽度时，块级元素宽度是它父级元素内容的宽度
  • 不设置高度时，块级元素高度是它本身内容的高度
  • 用于指定宽高，标签占满一行；

(3) 行内块元素

img, input, textarea

  • 本身属性为 display:inline-block
  • 不自动换行
  • 能够自动识别 width 和 height, line-height, padding, margin
  • 用于指定元素宽高，不占满一行

(4) 空元素

<br> <hr> <link> <meta>

页面导入样式时，使用link和@import有什么区别？

  • link是xhtml标签，除了加载css外，还可以定义RSS等其他事务；@import属于CSS范畴，只能加载CSS
  • link是xhtml标签，无兼容问题；@import是在css2.1提出来的，低版本的浏览器不支持
  • link支持使用javascript控制去改变样式，而@import不支持
  • link引用CSS时候，页面载入时同时加载；@import需要在页面完全加载以后加载，而且@import被引用的CSS会等到引用它的CSS文件被加载完才加载

无样式内容闪烁（FOUC）Flash of Unstyle Content

  • @import导入CSS文件会等到文档加载完后再加载CSS样式表
  • 在页面DOM加载完成到CSS导入完成之间会有一段时间页面上的内容是没有样式的
  • 使用link标签，这样页面会等到CSS下载完之后再下载HTML文件，这样先布局好，就不会出现FOUC问题。

介绍一下你对浏览器内核的理解？

  • 主要分成两部分：渲染引擎(Layout Engine或Rendering Engine)和JS引擎。
  • 渲染引擎：负责取得网页的内容（HTML、XML、图像等等）、整理讯息（例如加入CSS等），以及计算网页的显示方式
  • 然后会输出至显示器或打印机
  • 浏览器的内核的不同对于网页的语法解释会有不同，所以渲染的效果也不相同。
  • JS引擎：解析和执行javascript来实现网页的动态效果。
  • 最开始渲染引擎和JS引擎并没有区分的很明确，后来JS引擎越来越独立，内核就倾向于只指渲染引擎。

HTML5有哪些新特性,移除了那些元素？如何处理HTML5新标签的浏览器兼容问题？如何区分HTML和HTML5？

  • 新特性：新增了图像、位置、存储、多任务、拖放等功能
  1. 拖拽释放(Drag and drop)API
  2. 语义化更好的内容标签(header,nav,footer,aside,article,section)
  3. 音频，视频API(audio,video)
  4. 画布API(Canvas)
  5. 地理API(Geolocation)API
  6. 本地离线存储localStorage长期存储数据，浏览器关闭之后数据不丢失
  7. sessionStorage的数据在浏览器关闭之后自动删除
  8. 表单控件calendar,date,time,email,url,search
  9. 新的技术webworker,websocket,Geolocation
  • 移除元素
  1. 纯表现的元素：basefont big center font s strike tt u
  2. 性能较差元素：frame frameset noframes
  • 兼容问题处理
  1. 使用DOM操作来添加这些标签
  2. 使用封装好的 html5shiv.js 库
  • 如何区分

  1. 文档声明

  2. 结构语义

    • html：没有体现结构语义化的标签，通常都是<div id=”header”>这样来命名的；

    • html5：在语义上有很大优势，采用新的html5结构语义标签。

简述一下你对HTML语义化的理解？

  1. 去掉或丢失样式的时候能够让页面呈现出清晰的结构。
  2. 爬虫依赖于标签来确定上下文和各个关键字的权重。
  3. 有利于SEO和搜索引擎建立良好沟通，有助于爬虫抓取更多的信息，
  4. 方便其它设备解析。
  5. 便于团队开发和维护，语义化更具可读性。

HTML5的文件离线储存怎么使用，工作原理是什么？

  • 在线情况下，浏览器发现HTML头部有manifest属性，它会请求manifest文件
  • 如果是第一次访问，那么浏览器就会根据manifest文件的内容下载相应的资源，并进行离线存储。
  • 如果已经访问过并且资源已经离线存储了，那么浏览器就会使用离线的资源加载页面。
  • 然后浏览器会对比新的manifest文件与旧的manifest文件，如果文件没有发生改变，就不会做任何操作
  • 如果文件改变了，那么就会重新下载文件中的资源，并且进行离线存储

cookies sessionStorage localStorage 区别？

共同点：都是保存在浏览器端 且都是同源的

区别：

  • 用途不同

    • cookies 是为了标识用户身份而存储在用户本地终端上的数据 且始终在同源 http 请求中携带 即cookies是在浏览器和服务端来回传递
    • sessionStorage localStorage 仅在本地保存 不会自动把数据发送给服务端

  • 作用域不同：

    • cookies 在所有的同源窗口都共享
    • sessionStorage 在不同浏览器不共享
    • localStorage 在所有的同源窗口都共享

  • 存储大小限制不同

    • cookies 保存的数据很小 不超过 4K
    • sessionStorage localStorage 存储数据较大 可达 5M

  • 数据有效期不同：

    • cookies 在设置的 cookies 过期时间之内一直有效 即使窗口或者浏览器关闭，默认是会话关闭清除；

    • localStorage 如果不主动删除 始终有效 即使窗口或者浏览器关闭 用来长期存储本地数据

    • sessionStorage 仅在浏览器窗口关闭之前有效

iframe 框架有哪些优缺点？

优点：

  • iframe 能够原封不动的把嵌入的网页展现出来
  • 如果有多个网页引用 iframe 只需要修改 iframe 里面的内容 就可以实现调用的每一个页面的更改 方便快捷
  • 网页一般为了风格统一 头部和版面设计基本一致 可以写成一个页面 使用 iframe 嵌套 提高代码的可重用性
  • 如果遇到第三方加载缓慢的内容 如图标和广告 这些问题可以通过 iframe 解决

缺点：

  • 不利于 SEO 搜索引擎的爬虫程序 无法解读这种页面
  • 框架结构中会出现各种滚动条
  • 使用框架结构时 需要保证设置正确的导航链接
  • iframe 会增加服务器的 http 请求

label 的作用是什么？怎么使用？

作用：用于绑定一个表单元素, 当点击label标签的时候, 被绑定的表单元素就会获得输入焦点。

  • for：表示标签要绑定的 html 元素 一般需要绑定 id 名
  • accesskey：访问该 label 标签的热键 按下该热键 所绑定的元素获取焦点

HTML5 的 form 如何关闭自动完成功能？

html 输入框拥有自动完成功能：

  • 当往输入框输入内容时 浏览器会从之前的同名输入框历史记录中查找出类似的内容并列在输入框下面 不需要输入全部内容 只需要在下面选择输入内容即可
  • 但有时我们需要关闭输入框的自动完成功能(当用户输入内容的时候 希望使用 ajax 技术从数据库中搜索并列举 而不是从历史记录中查找类似内容)

关闭 form 自动完成功能的方法：

  • 在 IE 的 internet 选项菜单中的自动完成里面设置
  • 设置 form 输入框的 autocomplete 为 on 或 off 实现开启或关闭改功能

如何实现浏览器多个标签页之间的通信？

  • 使用 WebSocket SharedWorker
  • 也可以调用 localStorage cookies 等本地存储方式
  • localStorage 在另一个浏览器上下文里被添加 修改 或删除时 都会触发一个事件 我们通过监听事件 控制它的值来进行页面信息通信
  • 注意：Safari 在无痕模式下设置 localStorage 值时会抛出 QuotaExceededError 异常

页面可见性(Page Visibility) API 可以有哪些用途？

  • 通过 Visibility state 的值 检测页面当前是否可见 以及打开网页的时间
  • 在页面被切换到其他后台进程的时候 自动暂停音乐或视频的播放

如何在页面上实现一个圆形的可点击区域？

  1. border-radius
  1. js 不在圆上的算法

网页验证码是干嘛的？是为了解决什么安全问题？

  • 区分用户是计算机还是人的程序
  • 可以防止恶意破解密码、刷票、论坛灌水

title 与 h1 的区别、b 与 strong 的区别、i 与 em 的区别？

title 与 h1 的区别：

  • title 属性没有明确的意义 表示网页的标题
  • h1 表示层次明确的标题 对页面信息的抓取也有很大影响

b 与 strong 的区别：

  • b 是无意义的视觉表示
  • strong 标明重点内容
  • 视觉样式标签：b i u s
  • 语义样式标签：strong em ins del code

元素的 alt 和 title 有什么异同？

  • alt 作为图片的替代文字出现
  • title 是图片的解释文字

a 标签的 target 属性

_self _blank _parent _top

html5 的 datalist

和 input 标签配合使用，定义 input 可能的值，input 使用 list 绑定 datalist
