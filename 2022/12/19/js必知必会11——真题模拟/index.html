js内存泄露

js内存泄露定义：应用程序不再需要占用内存的时候，由于某些原因，内存没有被操作系统或可用内存池回收；

js内存泄露的原因：主要原因是一些不需要(不恰当)的引用；

常见的js内存泄露点：

  1. 闭包：闭包是有一个内部的代码区域，使用了外层某个区域的变量，那么在这个内部代码区域执行完成之前，外层的那个变量是不会被释放的；
  2. 定时器：setInterval 注意 this.timer = null; 并不能清除掉定时器，必须通过 clearInterval(timer) 才可以；
  3. DOM引用：避免DOM引用和闭包搅和在一起，还需要通过 btn = null; btn.onclick = null; 清除DOM事件；
  4. 未捕获的异常：什么时候打开控制台，都存在。

js事件循环和任务队列

  1. 宏任务和微任务

    任务队列：一个是宏任务队列，一个是微任务队列；

    宏任务：I/O、定时器、事件绑定、ajax

    微任务：Promise中的then、catch、finally；process.nextTick；async、await

  2. js事件循环：主线程执行完毕之后，从事件队列中去读取任务队列的过程，我们称为事件循环；

    • 所有同步任务都是在主线程上执行，形成一个执行栈；
    • 主线程之外还存在一个任务队列，只要存在异步任务，就会在任务队列里放置一个事件；
    • 一旦执行栈里面的同步任务代码执行完毕，主线程就会读取任务队列，结束等待状态，开始执行；
    • 主线程不断重复以上3个步骤。

ajax 请求 get 和 post 的区别

  • get 一般用于查询操作 post 一般用于用户提交操作
  • get 参数拼接在 url 上 post 放在请求体内 (数据体积可能更大)
  • 安全性：post 易于防止 XSRF

前端性能如何优化 一般从哪几方面考虑

  • 原则

    • 多使用内存、缓存或其他方法
    • 减少 CPU 计算量 减少网络加载耗时
    • 适用于所有编程的性能优化 —- 空间换时间

  • 方向

    • 让加载更快

      • 减少资源体积：压缩代码

      • 减少访问次数：合并代码和资源(Sprite图等) SSR 服务器端渲染 缓存

      • 使用更快的网络：CDN

    • 让渲染更快(顺畅)

      • CSS 放到 head 里 js 放到 body 下

      • 尽早开始执行 js 用 DOMContentLoaded 触发

      • 懒加载 (图片懒加载 上划加载更多)

      • 对 DOM 查询进行缓存

      • 频繁 DOM 操作 合并到一起插入 DOM 结构

      • 节流 throttle 防抖 debounce
